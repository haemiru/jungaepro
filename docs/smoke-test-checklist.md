# 중개프로 출시 전 스모크 테스트 체크리스트

> PPT 전국 발송 전에 **프로덕션(`jungaepro.com`)에서** 전체 흐름을 1회 직접 확인.
> 이번에 프로덕션 인프라를 크게 바꿨으므로(자동승인·메일·Edge Function 배포·SEO) 반드시 실환경 점검.

## A. 신규 가입 → 자동 승인 → 온보딩 (가장 중요)

- [ ] `jungaepro.com` → "무료로 시작하기" → 중개사(agent)로 회원가입 (테스트용 이메일)
- [ ] Supabase 이메일 인증 메일 수신 → 인증 링크 클릭
- [ ] **가입 알림 메일 2종 확인**:
  - [ ] junominu@gmail.com 에 "새 개공 가입" 알림 도착
  - [ ] 가입한 이메일에 "가입을 환영합니다 (자동 승인)" 도착 (발신 `noreply@jungaepro.com`)
- [ ] 로그인 → `{slug}.jungaepro.com/admin/dashboard`로 자동 이동되는지 (슬러그 미설정이면 admin으로)
- [ ] 슈퍼관리자 목록에서 방금 가입 계정이 **인증됨(자동)** 상태로 뜨는지

## B. 사무소 설정 → 포털 노출

- [ ] 환경설정 → 사무소 정보: 사무소명·주소·연락처 입력 + **서브도메인(slug)** 설정 → "사용 가능" → 저장
- [ ] `{slug}.jungaepro.com/` 접속 → 본인 사무소로 브랜딩된 사용자 포털이 뜨는지 (mock "스마트부동산" 아님)
- [ ] 포털 홈 하단 **부동산 뉴스**가 실제 기사로 뜨는지 (NAVER 연동)

## C. 매물 → 유저 노출

- [ ] 매물 관리 → 새 매물 등록 (사진 업로드 포함) → 저장
- [ ] 유저 포털 홈/검색에 방금 매물이 노출되는지
- [ ] 매물 상세 → **주변 실거래가**가 뜨는지 (MOLIT 국토부 연동)

## D. 문의 → 답변

- [ ] 유저 포털에서 매물 문의 or FloatingFAB 문의 접수 → 문의번호(INQ-…) 표시
- [ ] 관리자 → 문의 관리에 접수됨, 답변 작성 → 이메일 채널로 발송 → 고객 이메일 수신 확인
- [ ] (AI) 문의 답변 초안 생성 버튼 동작 (Gemini)
- [ ] 답변 임시저장 → 페이지 재방문 시 복원되는지

## E. 계약 → 트래커

- [ ] 계약 관리 → 새 계약 4단계 마법사(매물→양식→입력→미리보기) → 저장
- [ ] 계약 트래커에서 단계 체크·D-Day 표시
- [ ] 확인설명서 PDF 다운로드 동작

## F. 임장 (모바일 권장)

- [ ] 임장 관리 → 새 임장 → 체크리스트에서 항목 상태 + **사진 첨부** 동작 → 저장/완료 → 리포트

## G. SEO / 공유 (발송 직결)

- [ ] 카톡/문자로 `jungaepro.com` 링크 공유 → **미리보기 카드**(제목·설명·OG 이미지)가 뜨는지
  - 안 뜨면: 카톡은 캐시가 강해서 https://developers.kakao.com/tool/clear/og 로 캐시 초기화
- [ ] `jungaepro.com/robots.txt`, `jungaepro.com/sitemap.xml` 정상 응답
- [ ] `jungaepro.com/og-image.png` 이미지 정상

## H. 정리

- [ ] 테스트로 만든 가입 계정/매물/문의 정리 (또는 유지 결정)
- [ ] 문제 발견 시 → completion-plan.md에 기록 후 수정
